Section 01

Gitとは？

Gitは「ファイルの変更履歴を管理するツール」です。チームで同じファイルを編集しても、変更をきれいにまとめることができます。

📸

スナップショット管理

ファイルの状態を「セーブ」するように記録できます。過去の状態にいつでも戻れます。

🌿

ブランチ機能

「枝分かれ」を作って、元のファイルを壊さずに作業できます。

🤝

チーム開発

複数人が同時に異なる部分を編集し、後でひとつにまとめることができます。

🔐

履歴が残る

誰が・いつ・何を変えたかがすべて記録されます。ミスしても怖くありません。

💡

わかりやすいたとえ：
Gitは「Google ドキュメントの変更履歴」の超強化版です。変更のたびに「誰が・何のために変えたか」をメモ（コミットメッセージ）と一緒に保存できます。チームで使うときは GitHub というサービスを使ってクラウドで共有します。

Section 02

よく使う用語

最初は覚えるだけで大丈夫。使っていくうちに自然に身につきます。

repository

リポジトリ

プロジェクトのファイルと変更履歴をまるごと保存する「箱」。略して「リポ」とも呼ばれます。

commit

コミット

変更内容を記録する操作。「セーブ」のようなイメージ。メモ（コミットメッセージ）と一緒に保存します。

branch

ブランチ

作業を分岐させる機能。元のコードを壊さずに別の作業ができます。

merge

マージ

別ブランチでの変更を元のブランチに取り込む操作。ブランチを「合流」させます。

pull request

プルリクエスト（PR）

「自分の変更をマージしてください！」と提案する GitHub の機能。レビューと承認ができます。

clone

クローン

リモート（GitHub）にあるリポジトリを自分のパソコンにコピーする操作。

push

プッシュ

ローカルのコミットをリモート（GitHub）にアップロードする操作。

pull

プル

リモートの最新変更を自分のパソコンに取り込む操作。

Section 03

基本コマンド

ターミナル（コマンドプロンプト / Terminal）で入力するコマンドです。右のボタンでコピーできます。

リポジトリを作る
現在のフォルダを Git リポジトリとして初期化します
$ git init
リポジトリをコピーする
GitHub などにあるリポジトリを手元にダウンロードします
$ git clone https://github.com/ユーザー名/リポジトリ名.git
変更を記録する（add → commit）
ファイルを「ステージ」してからコミット（セーブ）します
$ git add ファイル名          # 特定のファイルをステージ
$ git add .               # 変更したファイルをすべてステージ
$ git commit -m "メッセージ"  # コミット（何を変えたか書く）
ブランチを作って切り替える
新しいブランチを作り、そこに移動します
$ git branch                  # ブランチ一覧を見る
$ git checkout -b ブランチ名   # 新ブランチを作って移動
アップロード / ダウンロード
ローカルとリモートの変更を同期します
$ git push origin ブランチ名   # リモートにアップロード
$ git pull                    # リモートの最新版を取得
状態・履歴を確認する
今の状態や過去のコミット履歴を見られます
$ git status         # 変更されたファイルを確認
$ git log --oneline   # コミット履歴をコンパクトに表示

Section 04

このサイトに参加する

自分の自己紹介ページをこのサイトに追加しましょう！5ステップで完了します。

1

リポジトリをクローンする

GitHub からプロジェクトを自分のパソコンにコピーします。

$ git clone https://github.com/ユーザー名/リポジトリ名.git $ cd リポジトリ名
2

ブランチを作る

自分専用のブランチを作って作業します。ブランチ名は自分の名前にしましょう。

$ git checkout -b yourname
3

自己紹介ページを作る

pages/template.html をコピーして自分のファイルを作り、内容を書き換えましょう。ファイル名は pages/yourname.html にします。

# Windows (コマンドプロンプト) copy pages\template.html pages\yourname.html # Mac / Linux cp pages/template.html pages/yourname.html
4

コミットしてプッシュする

変更を保存（コミット）して GitHub にアップロードします。

$ git add pages/yourname.html $ git commit -m "自己紹介ページを追加" $ git push origin yourname
5

Pull Request を作る

GitHub を開いて「Compare & pull request」ボタンをクリック。タイトルに 「〇〇の自己紹介を追加」 と書いて送信しましょう。管理者がレビューしてマージします！
