Section 01

インストール＆初期設定

まずは Git をパソコンに入れて、Cursor（VS Code）で使えるようにしましょう。

A
Git をインストールする
公式サイトからインストーラーをダウンロードします
  1. git-scm.com/downloads を開き、お使いの OS（Windows / Mac）用のインストーラーをダウンロードする
  2. インストーラーを起動し、すべてデフォルト（Next を押し続ける）でインストール完了
  3. インストール後、ターミナルで以下を実行してバージョンが表示されれば成功
$ git --version git version 2.xx.x ← こんな表示が出れば OK
B
名前とメールアドレスを登録する
コミットに「誰が変更したか」を記録するために必要です

ターミナルで以下を 1行ずつ 実行してください。
名前とメールは GitHub のアカウントと合わせるのがおすすめです。

$ git config --global user.name "山田 太郎" $ git config --global user.email "taro@example.com"
# 設定を確認するには： $ git config --global --list
C
Cursor（VS Code）で Git を使えるようにする
エディタから直接 Git 操作ができるようになります
  1. Cursor（または VS Code）を開き、左サイドバーの ソース管理アイコン（枝のマーク） をクリック
  2. フォルダを開いた状態で「リポジトリを初期化する」か、 クローンしたフォルダを開くと自動で Git が認識されます
  3. 変更したファイルが左サイドバーに表示される — M（Modified）や U（Untracked）のラベルで確認できます
  4. ファイル横の ＋ボタン で git add、 メッセージ入力→コミットボタン で git commit が完了します
💡
便利な拡張機能（任意）
拡張機能（Extensions）パネルで 「GitLens」 を検索してインストールすると、 各行に「誰がいつ変更したか」がエディタ上に表示されてとても便利です。
D
GitHub アカウントを Cursor / VS Code に連携する
プッシュ・プルを GUI で行えるようになります
  1. github.com でアカウントを作成（まだの人）
  2. Cursor のソース管理パネル下部にある 「GitHub にサインイン」 をクリック （または左下のアカウントアイコンから）
  3. ブラウザが開くので GitHub アカウントでログイン → 認証完了
  4. これで git push / git pull を GUI ボタンや コマンドパレット（Ctrl+Shift+P）から実行できます
⌨️
コマンドパレットからの操作
Ctrl+Shift+P（Mac: ⌘+Shift+P）を押して Git: Clone や Git: Push と入力すると、 コマンドを探さなくてもすばやく操作できます。

Section 02

Gitとは？

Gitは「ファイルの変更履歴を管理するツール」です。チームで同じファイルを編集しても、変更をきれいにまとめることができます。

📸

スナップショット管理

ファイルの状態を「セーブ」するように記録できます。過去の状態にいつでも戻れます。

🌿

ブランチ機能

「枝分かれ」を作って、元のファイルを壊さずに作業できます。

🤝

チーム開発

複数人が同時に異なる部分を編集し、後でひとつにまとめることができます。

🔐

履歴が残る

誰が・いつ・何を変えたかがすべて記録されます。ミスしても怖くありません。

💡

わかりやすいたとえ：
Gitは「Google ドキュメントの変更履歴」の超強化版です。変更のたびに「誰が・何のために変えたか」をメモ（コミットメッセージ）と一緒に保存できます。チームで使うときは GitHub というサービスを使ってクラウドで共有します。

Section 04

基本コマンド

ターミナル（コマンドプロンプト / Terminal）で入力するコマンドです。右のボタンでコピーできます。

リポジトリを作る
現在のフォルダを Git リポジトリとして初期化します
$ git init
リポジトリをコピーする
GitHub などにあるリポジトリを手元にダウンロードします
$ git clone https://github.com/ユーザー名/リポジトリ名.git
変更を記録する（add → commit）
ファイルを「ステージ」してからコミット（セーブ）します
$ git add ファイル名          # 特定のファイルをステージ
$ git add .               # 変更したファイルをすべてステージ
$ git commit -m "メッセージ"  # コミット（何を変えたか書く）
ブランチを作って切り替える
新しいブランチを作り、そこに移動します
$ git branch                  # ブランチ一覧を見る
$ git checkout -b ブランチ名   # 新ブランチを作って移動
アップロード / ダウンロード
ローカルとリモートの変更を同期します
$ git push origin ブランチ名   # リモートにアップロード
$ git pull                    # リモートの最新版を取得
状態・履歴を確認する
今の状態や過去のコミット履歴を見られます
$ git status         # 変更されたファイルを確認
$ git log --oneline   # コミット履歴をコンパクトに表示

Section 05

このサイトに参加する

自分の自己紹介ページをこのサイトに追加しましょう！5ステップで完了します。

1

リポジトリをクローンする

GitHub からプロジェクトを自分のパソコンにコピーします。

$ git clone https://github.com/ユーザー名/リポジトリ名.git $ cd リポジトリ名
2

ブランチを作る

自分専用のブランチを作って作業します。ブランチ名は自分の名前にしましょう。

$ git checkout -b yourname
3

自己紹介ページを作る

pages/template.html をコピーして自分のファイルを作り、内容を書き換えましょう。ファイル名は pages/yourname.html にします。

# Windows (コマンドプロンプト) copy pages\template.html pages\yourname.html # Mac / Linux cp pages/template.html pages/yourname.html
4

コミットしてプッシュする

変更を保存（コミット）して GitHub にアップロードします。

$ git add pages/yourname.html $ git commit -m "自己紹介ページを追加" $ git push origin yourname
5

Pull Request を作る

GitHub を開いて「Compare & pull request」ボタンをクリック。タイトルに 「〇〇の自己紹介を追加」 と書いて送信しましょう。管理者がレビューしてマージします！
